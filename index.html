Dokumentation av dethär APIet


Routs

GET/users - returnerar alla användare

GET/users/{id} - returnerar en användare med specifikt id

POST/users - skapar en ny användare och accepterar JSON object på formatet {"username", "password"}

PUT/users/{id} - ändrar en befintlig användare med specifikt id och accepterar JSON object med formatet {"username", "password"}

POST/login - tar JSON object med formatet {"username", "password"} och jämför det med databasen, om en användare med samma username och password hittas skickas ett fint välkomsmeddelande
